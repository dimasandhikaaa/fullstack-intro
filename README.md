# Fullstack Introduction Lab

## Overview

This project is part of the **MongoDB MERN Fullstack Developer Program**.
The purpose of this lab is to introduce the **basic concepts of Full Stack Development and Web Application Architecture**.

In this exercise, we explore how different parts of a web application communicate with each other, including the **client, frontend, backend, and database**.

---

## Learning Objectives

After completing this lab, students are expected to:

* Explain the concept of **Full Stack Development**
* Differentiate between **Frontend** and **Backend**
* Understand **Client–Server Architecture**
* Describe the role of **MongoDB in MERN Stack applications**

---

## Client–Server Architecture

Modern web applications typically follow a **client–server architecture**.

### Architecture Flow

User
↓
Browser (Client)
↓
Frontend (UI Layer)
↓
Backend (Server)
↓
Database (MongoDB)

### Components Explanation

**Client (Browser)**
The client is the platform used by users to access the application, such as Chrome, Firefox, or Edge.

**Frontend (UI Layer)**
The frontend is responsible for displaying the user interface and handling user interactions.
Common technologies include **HTML, CSS, JavaScript, and React**.

**Backend (Server Layer)**
The backend processes requests from the frontend, handles application logic, and communicates with the database.
Typical technologies include **Node.js and Express.js**.

**Database (MongoDB)**
MongoDB is a **NoSQL database** used to store application data such as user information, products, and other records.

---

## MERN Stack Overview

The **MERN Stack** is a JavaScript-based technology stack used to build full stack web applications.

It consists of:

* **MongoDB** – NoSQL database used to store data
* **Express.js** – Backend framework for Node.js
* **React.js** – Frontend library for building user interfaces
* **Node.js** – JavaScript runtime for running server-side code

One advantage of the MERN stack is that **JavaScript is used across the entire application**, from frontend to backend.
